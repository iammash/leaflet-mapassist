  • Limitations

  • Projection

    Currently, the only projection accepted is the default one: EPSG 3857. All data must be in this projection for it to work in the application.

  • Shapefiles

    If you have a shapefile within ArcMap or QGIS you can export the feature as a JSON to work with Leaflet Map Assist.

  • Geocoder

    Using free Geocoder plugin which uses Nomatim geocoder. Not to be relied upon for accuracy, but good for getting map to places of interest.

  • Data Size

    Without the use of a server, the resulting map and all the assets will be downloaded on the loading of the site. Therefore pruning data and properties of data can benefit the map performance and load time. Also, loading super large datasets will crash the application.

  • Local File Loading

    Data parser is using a local file I/O to load the data.

  • Additional Info:

    For tooltips and popup forms you can add layer property variables. ie. A dataset with property of species (layer.properties.species) can be used by adding dollar sign and property name -> $species. When using $ symbol for currency add an extra $ symbol as it wil escape and return $ symbol and the currency value instead of trying to find property with name of currency value.
